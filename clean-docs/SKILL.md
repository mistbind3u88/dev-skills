# clean-docs

`.claude/docs` 配下のタスクドキュメントを整理する。

## 前提

- `.claude/docs` にはタスク単位の作業ドキュメントが配置されている
- タスクが完了しmainにマージされたら、ドキュメントは削除対象になる
- 残す価値のある内容（仕様・設計判断・運用知識など）があれば、リポジトリ管理のAGENTS.mdやSKILL.mdに移行してから削除する

## 手順

### 1. 対象ドキュメントの一覧を取得する

worktree内で作業中の場合は元リポジトリの `.claude/docs` を対象とする。

```bash
git worktree list
ls -la .claude/docs/
```

worktreeの場合は元リポジトリのパスを特定し、そちらの `.claude/docs` を対象にする。

`.claude/docs` が存在しない、または空の場合は「整理対象のドキュメントはありません」と報告して終了する。

### 2. 各ドキュメントの内容を確認する

各ファイル（ディレクトリの場合はその中のファイルすべて）を読み、以下を把握する:

- ドキュメントの目的・対象タスク
- 関連するブランチ名やPR番号（記載があれば）
- 内容のうち、既存のAGENTS.md / SKILL.mdに反映されていない仕様・設計知識があるか

### 3. マージ状況を確認する

ドキュメントに関連するブランチやPRが特定できる場合は、マージ済みか確認する。

```bash
git log --oneline main --grep="<関連キーワード>" | head -5
git branch --merged main
```

特定できない場合は、ドキュメントの最終更新日時とmainの履歴を照合して判断材料とする。

### 4. ユーザーに整理方針を提示する

各ドキュメントについて以下のいずれかを提案する:

- **削除**: タスク完了済みで、内容がすでにAGENTS.md / SKILL.mdに反映済み、または一時的な作業メモのみ
- **移行して削除**: 残す価値のある仕様・設計知識があり、AGENTS.mdやSKILL.mdに反映してから削除
- **保留**: マージ状況が不明、または進行中のタスクに関連する可能性がある

一覧をまとめてユーザーに確認を取る。**ユーザーの承認なしに削除や移行を実行しない。**

### 5. 承認された方針にしたがって実行する

#### 移行する場合

1. 移行先のAGENTS.mdまたはSKILL.mdを読み、現在の構成を把握する
2. ドキュメントから移行対象の内容を抽出し、移行先の構成に合わせて整形する
3. 移行先に追記・更新する
4. 元のドキュメントを削除する

#### 削除のみの場合

```bash
rm <対象ファイル>
```

ディレクトリ内の全ファイルを削除した場合は空ディレクトリも削除する。

```bash
rmdir <対象ディレクトリ>
```

### 6. 結果を報告する

実行した内容をまとめて報告する:

- 削除したドキュメント
- 移行した内容と移行先
- 保留したドキュメントとその理由

## 注意

- `$ARGUMENTS` が指定されていれば、それを対象ドキュメントのパスまたはキーワードとして絞り込みに使う
- 移行先のAGENTS.md / SKILL.mdを編集する際は、既存の構成・スタイルに合わせる
- 実装上残していない仕様への言及は移行しない
