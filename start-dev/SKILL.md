# start-dev スキル

新しい作業を開始する準備を行う。ブランチ作成、worktree セットアップ、Issue/PR の情報取得、プランニングをまとめて実施する。

## 引数

```
$ARGUMENTS: <開発内容>
```

引数は**ブランチを切って対応する開発内容**として解釈する。以下の形式を受け付ける:

- **Issue/PR 番号**（例: `#123`, `123`）: カレントリポジトリの Issue/PR として `/gh-read` で情報を取得する
- **クロスリポジトリ参照**（例: `owner/repo#123`）: 別リポジトリの Issue/PR として `/gh-read` で情報を取得する
- **ブランチ名**（例: `f/add-feature`）: そのまま使用する。`/` を含む場合にブランチ名と判断する
- **自由記述**（例: `棚卸し手順のskill化`）: 開発内容の説明として扱い、ブランチ名を自動生成する

引数が何であっても、このスキルの目的は**作業ブランチの作成と開発開始の準備**である。引数の内容を実行指示として解釈してはならない。

## 手順

### 1. 前提確認

```bash
git status -s
git rev-parse --abbrev-ref HEAD
```

- 未コミットの変更がある場合はコミットを促して停止する
- 現在のブランチを確認する
- **作業ブランチにいる場合**（main/master 以外）: スキル `/takeover` に委譲して終了する

### 2. Issue/PR の情報取得

`$ARGUMENTS` に Issue/PR 番号が含まれる場合、スキル `/gh-read` を実行して情報を取得する。

取得した情報は以降のブランチ名生成と作業概要の提示に使用する。

### 3. ブランチ名の決定

- **カレントリポジトリの Issue/PR**: タイトルからブランチ名を自動生成する
  - 形式: `f/<slug>-<number>`（例: `f/add-user-auth-123`）
  - slug はタイトルを小文字化し、英数字とハイフンのみに変換、30文字以内に切り詰める
- **別リポジトリの Issue/PR**: リポジトリ名を含めてブランチ名を生成する
  - 形式: `f/<repo>-<slug>-<number>`（例: `f/other-repo-fix-auth-456`）
- **自由記述の場合**: 開発内容からブランチ名を自動生成し、ユーザーに確認する
  - 日本語の場合は内容を英語の slug に変換する（例: `棚卸し手順のskill化` → `f/tanaoroshi-skill`）
- **ブランチ名が直接指定された場合**: そのまま使用する

### 4. main/master を最新化して worktree 作成

```bash
git switch main
git pull --ff-only origin main
git worktree add <worktree パス> -b <ブランチ名>
```

main への pull が fast-forward できない場合はユーザーに報告して停止する。

worktree のパスはユーザーに確認する。

### 5. 既存タスクドキュメント確認

元ディレクトリの `.claude/docs` 配下に関連するタスクドキュメントがあるか確認する。関連するドキュメントがあれば内容を提示する。

このステップは**既存ドキュメントの参照のみ**。新規作成はしない（step 6 参照）。

### 6. プランニング

次セッションへのハンドオーバーのため、**プランモードに入り実装計画を必ず作成する**。プランは `~/.claude/plans/` 配下に保存され、Claude Code の `/takeover` や `/respond` 等から参照できる。

プランには以下を含める:

- 変更対象のファイル・関数
- 変更の方針（何をどう変えるか）
- テスト方針（どう検証するか）
- 関連 Issue への参照（常に `owner/repo#123` 形式で記載する）

対応内容が曖昧で実装計画を立てられない場合は、判明している情報と次セッションで調査すべき事項をプランに記載する。

#### タスクドキュメントは原則作らない

Claude Code 内で作業する限り、ハンドオーバー先はプランで十分である。`.claude/docs` 配下のタスクドキュメントは**他のコーディングエージェント（Codex 等）と協業する場合のデフォルト配置先**という位置づけで、Claude Code 単独の開発では作成しない。

プランの複製、プランへのポインタのみのドキュメント、背景情報の二重記載は禁止する。

### 7. 作業概要の提示

以下を表示して作業開始の準備完了を報告する:

- 作成したブランチ名と worktree パス
- Issue/PR の概要（取得した場合）
- 参照した既存タスクドキュメント（存在する場合）
- 実装計画

Claude Code のセッションは起動ディレクトリに紐づくため、worktree での作業はユーザーに新しいセッションの起動を案内する。

## 注意

- このスキルはブランチ作成・情報取得・プランニングまでを行う。コードの変更は行わない
- Issue/PR の作成・更新はスキル `/gh-edit` で行う
