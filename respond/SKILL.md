# respond スキル

レビュー指摘への対応を一連の手順で実行する。

## 手順

以下のスキルを順番に実行する。いずれかが失敗したら停止し、どのステップで止まったかを報告する。

### 1. `/collect-feedback`

PR のレビューコメントを収集し、要対応事項を整理する。

### 2. `/fixup`

要対応の指摘に対してコードを修正し、fixup コミットを作成する。

複数の指摘がある場合は、関連する指摘をまとめて1つの fixup にするか、個別に fixup を作成するかをユーザーに確認する。

### 3. `/ship --skip-review --skip-gh-edit`

commit（autosquash）→ check → push を実行する。review と gh-edit はスキップする（指摘対応で概要欄の更新やレビューは不要）。

### 4. `/reply-review`

対応した指摘にリプライを投稿する。

## 注意

- 各ステップの詳細は対応するスキルの SKILL.md に従う
- ステップ1で要対応の指摘がない場合はワークフロー全体を終了する
- ステップ2で修正が不要と判断した指摘は、ステップ4で対応不要の理由をリプライする
- `/check` の指摘に対応して新たにコミットを作成した場合は、ship スキルのルールに従い push せずにワークフローを中断する
