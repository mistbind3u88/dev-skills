# ship スキル

変更を仕上げて送り出すまでの一連の手順を実行する。

## 手順

以下のスキルを順番に実行する。いずれかが失敗したら停止し、どのステップで止まったかを報告する。

### 1. `/check`

未チェック項目（lint・build・test・review）を実行する。

### 2. `/commit`

変更をコミットする。

### 3. `/push`

チェック済みであることを確認して push する（ステップ1で通過済みのためスキップが期待される）。

### 4. `/gh-edit` と `/watch-ci` を並行実行する

push 完了後、以下を並行で実行する:

- `/gh-edit` — PR の概要欄を作成・更新する
- `/watch-ci` — GitHub Actions の CI 実行を監視する

`/watch-ci` が失敗を報告した場合、その旨をユーザーに伝える。

## 注意

- 各ステップの詳細は対応するスキルの SKILL.md に従う
- ステップ間でユーザーの判断が必要になった場合（レビュー指摘への対応、コミットメッセージの確認など）、該当スキルの中でユーザーとやり取りしてから次に進む
- `$ARGUMENTS` で `--skip-review` が指定された場合は `/check --skip-review` に渡す
- コミットする変更がない場合はステップ2をスキップする
- `/check` の指摘に対応して新たにコミットを作成した場合は、push せずにワークフローを中断する。チェック通過後に追加変更が入ると、CI 上で不整合が検知されるリスクがあるため、再度 `/ship` を実行してもらう
