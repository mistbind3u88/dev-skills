# doc-check スキル

コード変更に対して関連ドキュメントが追従しているかをチェックし、結果を報告する。修正は行わない。

**検査は必ず `general-purpose` サブエージェントに委譲する。** 書き手（直前に実装した本人）と検査者を分離することで、自己レビューでは見落とされがちな「コードの逐語説明になっていないか」「実装詳細の羅列になっていないか」を独立した視点で判定させる。メイン側は起動・結果受領・報告のみを担う。

## 手順

### 1. サブエージェントを起動する

`general-purpose` サブエージェントを `run_in_background: true` で起動する。同期実行にしない。コンソールを占有せず、完了通知を受け取ってから結果を読む。

#### サブエージェントへの入力

プロンプトに以下を含める。

- 検査対象: 現在のリポジトリ（作業ディレクトリ）
- 比較元ブランチ: 原則 `main`（リポジトリの既定ブランチが異なる場合はそれを使う）
- 参照すべき判定基準のファイル: `doc-check/AGENTS.md`（スキル配下に存在する場合）
- 対象ドキュメント種別
  - `AGENTS.md` / `CLAUDE.md` — リポジトリ全体およびスキル固有の規則・制約
  - `**/SKILL.md` — スキル定義（手順・allowed-tools・description）
  - `README.md` — プロジェクト概要
- 観点（下表）とチェック内容
- 出力フォーマット（後述）

#### 検査観点

| 観点                   | チェック内容                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| ---------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 参照の有効性           | ドキュメントが参照しているファイル・関数・コマンドが存在するか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| 手順の正確性           | SKILL.md に記載された手順が現在のコード・スクリプトと一致するか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| allowed-tools の過不足 | スキルが使用するコマンドが `allowed-tools` に含まれているか、不要なものが残っていないか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| description の正確性   | スキルの `description` が現在の機能を正しく反映しているか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| 削除済みコードへの言及 | 削除されたファイル・関数・設定への参照が残っていないか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| レビュー誤指摘の予防   | コードを読むだけでは誤りに見える設計判断（暗黙の副作用への依存、意図的な省略、他リポジトリとの整合など）がコメントやドキュメントで説明されているか                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| 記述スタイル           | **特に重点的に検査する**。`AGENTS.md` / `CLAUDE.md` および `SKILL.md` の散文部分がコードの逐語説明や実装詳細の羅列になっていないか。段落ごとに「コードを読んでも得られない情報が含まれているか」を問い、含まれていなければ要修正として扱う。具体的な要修正パターン: (1) バッククォートで囲まれた識別子（関数名・型名・ファイル名）が 1 段落に 3 つ以上連続する、(2)「〜関数は〜を呼び出して〜を返す」「〜ファイルに〜を追加した」のような差分・呼び出し関係のパラフレーズ、(3) ディレクトリ構成図やファイル一覧を貼っているが、その構造を前提とした判断基準が書かれていない、(4) コード断片を引用しているが、その引用が「なぜこれを選んだか」の説明になっていない。判定基準の詳細は `doc-check/AGENTS.md` の「書くこと / 書かないこと / コンパクトに保つ」を参照。`SKILL.md` の手順セクションは「手順の正確性」観点で扱い、この観点の対象外 |

#### サブエージェントの出力フォーマット

観点ごとに `OK` / `要修正` を示し、`要修正` の項目は該当ファイルと抜粋・修正方針を添えて報告させる。自由記述の所感は不要。

```
観点別結果:
  参照の有効性:           OK
  手順の正確性:           要修正
    - commit/SKILL.md 手順2b: autosquash の起点指定例が現在のスクリプトと合っていない
  allowed-tools の過不足: OK
  description の正確性:   OK
  削除済みコードへの言及: OK
  レビュー誤指摘の予防:   OK
  記述スタイル:           要修正
    - tanaoroshi/CLAUDE.md「サブコマンド」節: 表がコードの挙動をそのまま写しており、選択基準・制約が書かれていない
```

### 2. 結果を報告する

サブエージェントの完了通知を受けたら出力を読み、ユーザーに報告する。

- 全観点 `OK` の場合: 成功として報告する
- `要修正` がある場合: 失敗として一覧を報告する

```
ドキュメント整合性チェック:
  commit/SKILL.md:    allowed-tools に不要な Bash(git branch:*) が残っている
  tanaoroshi/CLAUDE.md: 記述スタイル — コード挙動のパラフレーズに留まっている
```

## 注意

- このスキルはチェックと報告のみを行う。ドキュメントの修正はスキル `/doc-sync` で行う
- `/check` から呼び出された場合、不整合があれば check 全体を失敗にする
- サブエージェントは必ずバックグラウンドで起動する。同期実行にしない
- 要修正対応後の再検査は、呼び出し側が `/doc-check` を再度実行することで行う（このスキル内でループしない）
