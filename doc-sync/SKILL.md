# doc-sync スキル

コード変更に対して関連ドキュメントが追従しているかをチェックし、不整合があれば修正する。

## 手順

### 1. 変更内容を把握する

```bash
git diff --name-only main..HEAD
git log --oneline main..HEAD
```

変更されたファイルの一覧とコミット履歴を確認する。

### 2. 関連ドキュメントを特定する

変更内容から、影響を受ける可能性のあるドキュメントを特定する。

対象ドキュメント:

- `AGENTS.md` / `CLAUDE.md` — リポジトリ全体の規則・制約
- `**/SKILL.md` — スキル定義（手順・allowed-tools・description）
- `**/AGENTS.md` — スキル固有の補足指示
- `README.md` — プロジェクト概要

特定の基準:

- 変更されたファイルと同じディレクトリのドキュメント
- 変更内容が言及されているドキュメント（Grep で検索）
- スキルの allowed-tools や手順に影響するコード変更

### 3. 不整合をチェックする

各ドキュメントを読み、以下の観点で不整合がないか確認する。

| 観点                   | チェック内容                                                                          |
| ---------------------- | ------------------------------------------------------------------------------------- |
| 参照の有効性           | ドキュメントが参照しているファイル・関数・コマンドが存在するか                        |
| 手順の正確性           | SKILL.md に記載された手順が現在のコードと一致するか                                   |
| allowed-tools の過不足 | スキルが使用するコマンドが allowed-tools に含まれているか、不要なものが残っていないか |
| description の正確性   | スキルの description が現在の機能を正しく反映しているか                               |
| 削除済みコードへの言及 | 削除されたファイル・関数・設定への参照が残っていないか                                |

### 4. 結果を報告する

不整合が見つかった場合、一覧で報告する:

```
ドキュメント整合性チェック:
  commit/SKILL.md:  allowed-tools に不要な Bash(git branch:*) が残っている
  check/SKILL.md:   ステップ2a の例が現在のプロジェクト構成と合っていない
```

### 5. 修正を実施する

ユーザーの承認を得てから修正する。

- ドキュメントの更新: Edit で修正
- 不要なドキュメントの削除: ユーザーに確認してから削除
- 実装上残していない仕様への言及は削除する

## 注意

- このスキルはドキュメントの整合性チェックと修正を行う。コードの変更は行わない
- `/check` から呼び出された場合、不整合が見つかったら失敗として報告する（修正はユーザーの指示後）
- `clean-docs` との違い: `clean-docs` はタスクドキュメント（`.claude/docs`）の整理、`doc-sync` はコード変更に対するドキュメントの追従チェック
