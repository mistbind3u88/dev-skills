# doc-sync スキル

`/doc-check` で検出された不整合に基づいて、関連ドキュメントを更新・削除する。

## 手順

### 1. 不整合を特定する

スキル `/doc-check` を実行して、現在の不整合を把握する。

### 2. 修正方針をユーザーに提示する

各不整合について修正方針を提案する:

- ドキュメントの更新（記述の修正・追記）
- 不要な記述の削除
- 実装上残していない仕様への言及の削除
- レビュー誤指摘を防ぐためのコードコメント追加（非自明な設計判断の説明）

一覧をまとめてユーザーに確認を取る。**ユーザーの承認なしに修正を実行しない。**

### 3. 修正を実施する

承認された方針に従い、Edit でドキュメントを修正する。

### 4. 再チェック

修正後にスキル `/doc-check` を実行し、不整合が解消されたことを確認する。

## 注意

- このスキルはドキュメントの修正を行う。コードの変更は行わない
- チェックのみを行いたい場合は `/doc-check` を使う
- `clean-docs` との違い: `clean-docs` はタスクドキュメント（`.claude/docs`）の整理、`doc-sync` はコード変更に対するドキュメントの追従修正
