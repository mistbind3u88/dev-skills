# issue-review — Issue 有効性分析スキル

古い GitHub Issue を分析し、コードベースの現状と照合して有効性を判定するエージェント駆動スキル。

## 分析のアプローチ

- Issue の body とコメントからコード参照を抽出する際、バッククォート内のテキスト、ファイルパス形式の文字列、GitHub URL を対象とする
- 関数名やクラス名は Grep で検索する。完全一致でヒットしない場合はリネームの可能性を考慮する
- git log の --since は Issue の createdAt を使う。Issue 作成前の変更は調査対象外
- 4軸評価は根拠を明示する。「おそらく」「たぶん」は避け、コードの状態から客観的に判断する

## 既知の制約

### リポジトリのクローンが必要

git log や Grep によるコード調査はローカルにクローンされたリポジトリでのみ動作する。対象リポジトリがローカルにない場合は、gh api でできる範囲の調査に留め、その旨を報告に記載する。

### コメントの取得上限

gh issue view の --json comments は最新 100 件のみ取得する。それ以上のコメントがある Issue では gh api でページネーションが必要。
