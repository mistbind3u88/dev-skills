# Issue 有効性分析

Issue を現在のコードベースと照合し、問題がまだ有効か、対応する価値が残っているかを判定する。

## 入力

引数で対象 Issue を指定する。

```
/issue-review owner/repo#123
/issue-review #123
/issue-review owner/repo#123 owner/repo#456
```

`#123` のみの場合はカレントリポジトリの Issue として扱う。複数指定時は順に分析する。

## 出力

Issue ごとに構造化された分析レポートを会話に出力する。

## 手順

### Phase 1: Issue の詳細取得

```bash
gh issue view <number> --repo <owner/repo> --json title,body,author,createdAt,updatedAt,comments,labels,assignees
```

- タイトル、本文、作成日、コメント履歴を把握する
- 本文からコード参照を抽出する:
  - GitHub URL（`https://github.com/owner/repo/blob/...#L123`）
  - ファイルパス（`pkg/foo/bar.go`）
  - 関数・型・変数名（コード中の識別子）

### Phase 2: コード参照の現存確認

Phase 1 で抽出したコード参照を検証する。

1. **ファイルパス・URL**: 言及されているファイルが存在するか（Glob で確認）
2. **関数・型・変数名**: コードベースに存在するか（Grep で検索）
3. **GitHub URL 内の行番号**: 該当行の内容が Issue 作成時の記述と一致するか（Read で確認）

存在しない参照は「削除済み・リネーム済み」として記録する。

### Phase 3: Issue 作成後の変更履歴

参照されているファイルに対して、Issue 作成日以降のコミットを確認する。

```bash
git log --oneline --since=<createdAt> -- <file>
```

- Issue の問題を直接解決したと思われるコミットがあるか
- 関連する大規模なリファクタリングがあったか
- 気になるコミットがあれば `git show <hash>` で内容を確認する

### Phase 4: 現在のコード状態の分析

- 該当コードを Read で読み、Issue が主張する問題が現在も成立するか確認する
- 利用箇所（呼び出し元）を Grep で特定し、影響範囲を把握する
- Issue の「対策案」が記載されている場合、その方針が現在のコードに適用済みかを確認する

### Phase 5: 判定

以下の4軸で判定する。各判定に根拠（コード参照、コミットハッシュ等）を添える。

| 観点             | 選択肢                                   |
| ---------------- | ---------------------------------------- |
| 問題は現存するか | はい / いいえ / 部分的                   |
| 実害が出ているか | はい / いいえ / 潜在的                   |
| 取り込む価値     | 高 / 中 / 低                             |
| 推奨アクション   | 修正 / クローズ / 保留 / 別 Issue に統合 |

## 出力形式

```markdown
## Issue 分析: owner/repo#123

### 概要

| 項目     | 内容                  |
| -------- | --------------------- |
| タイトル | ...                   |
| 作成者   | ...                   |
| 作成日   | ...                   |
| 経過     | N年Mヶ月、コメントN件 |

### Issue の主張

（Issue の問題を簡潔に要約）

### コード参照の現存確認

| 参照                 | 状態                 |
| -------------------- | -------------------- |
| `pkg/foo/bar.go:123` | 存在（内容変更あり） |
| `FunctionName()`     | 削除済み             |

### Issue 作成後の変更履歴

（関連コミットの要約）

### 現在のコードの状態

（Issue の前提が成立するかの分析）

### 判定

| 観点             | 判定 |
| ---------------- | ---- |
| 問題は現存するか | ...  |
| 実害が出ているか | ...  |
| 取り込む価値     | ...  |
| 推奨アクション   | ...  |

（各判定の根拠）
```

## 注意

- Issue への言及は `owner/repo#N` 形式で記載する
- 判定は根拠に基づいて行い、不明な点は不明と記載する
- Issue のクローズや編集は行わない。判定結果の報告のみ
