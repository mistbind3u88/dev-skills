# gh-edit

## 前提ツール

- [git](https://git-scm.com/)
- [gh](https://cli.github.com/) — `brew install gh`

## PR概要欄のスタイル

PR概要欄の作成・更新時は `pull_request_template.md` をベースに使う。以下のスタイル規則に従うこと。

### 全般

- テンプレート内の見出し（`# Subject`、`## やったこと` 等）はリテラルな固定文字列であり、置き換えやパラフレーズをしない
- **日本語**で記述する
- **変更規模に合わせる**: 小さい変更に大げさな構造を使わない。1〜2文で済むなら見出し構造なしでよい
- **重複を避ける**: 同じ情報をセクション間で繰り返さない。関連リンク先の内容も再説明しない。各セクションは固有の情報だけを書き、他で述べた内容は参照に留める

### `# Subject`

- 背景や目的を簡潔に説明する
- 複雑な変更やパフォーマンス改善では原因→解決策→効果の構造で説明する
- issue, 別PR, Slackスレッドなど背景の参照先があれば `- ref: <リンク>` の箇条書きで記載する

### `## やったこと`

- PRの主目的の達成に向けて重要な変更と意思決定を記載する
- 細部の実装都合は概要欄ではなくGitHub上のコメントやコード上のコメントで補足する

### `## やってないこと`

- 関連するがこのPRでは対応しないことがあれば、なぜこのPRでやらないのかの意思決定を含めて明記する
- なければセクションごと削除する

### `## 確認したこと`

- 変更内容に対して、CIでは自動検証されない確認事項がないか能動的に洗い出す
- 確認すべき項目があれば未チェック状態（`- [ ]`）で列挙する（GraphQLレスポンス、スクリーンショット、SQL実行結果、パフォーマンス比較など）
- ビルド・テスト・lintなどCIで検証される項目は書かない
- なければセクションごと削除する
