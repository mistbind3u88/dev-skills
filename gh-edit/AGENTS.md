# gh-edit

## 前提ツール

- [git](https://git-scm.com/)
- [gh](https://cli.github.com/) — `brew install gh`

## PR 概要欄のスタイル

PR 概要欄の作成・更新時は `TEMPLATE.md` をベースに使う。以下のスタイル規則に従うこと。

- **日本語**で記述する
- **スコープ外の明示**: 関連するがこのPRでは対応しないことがあれば明記する（例: 「〜の対応は別PRで実施する」）
- **関連リンク**: issue, 別PR, Slackスレッドなど背景の参照先があれば冒頭で記載
- **確認事項**: 動作確認した内容があれば記載（GraphQLレスポンス、スクリーンショット、SQL実行結果、パフォーマンス比較など）
- **変更意図**: 複雑な変更やパフォーマンス改善では原因→解決策→効果の構造で説明
- **変更規模に合わせる**: 小さい変更に大げさな構造を使わない。1〜2文で済むなら見出し構造なしでよい
